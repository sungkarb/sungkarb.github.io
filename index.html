01

Policies that balance latency & throughput

per subsystem
Scheduler
CFS — Completely Fair Scheduler
Weighted fair-share via virtual runtime (vruntime). Preempts on wakeup if the woken task has a sufficiently smaller vruntime. Tunable via sched_latency_ns and sched_min_granularity_ns.
latency fairness throughput
Scheduler
EEVDF (v6.6+)
Earliest Eligible Virtual Deadline First — replaces CFS preemption heuristics with a deadline-based pick. Reduces latency variance while preserving throughput.
latency deadline
Scheduler
SCHED_FIFO / SCHED_RR
Real-time policies. FIFO runs until block/yield; RR adds a timeslice. Hard latency guarantees at the cost of starvation risk for non-RT tasks.
latency real-time
I/O
mq-deadline
Assigns per-request deadlines. Batches requests within a window (fifo_expire_*) to favour throughput, but guarantees deadline expiry for latency-sensitive I/O.
deadline throughput
I/O
BFQ — Budget Fair Queuing
Assigns budgets (sectors) proportionally. Achieves low latency for interactive workloads by tracking idle state and serialising on the "in-service entity" tree.
latency fairness
I/O
Kyber
Lightweight scheduler for NVMe/fast storage. Maintains separate read/write token buckets; adjusts queue depth to hit configurable latency targets.
latency throughput
Memory
NUMA Balancing
Periodically scans and migrates pages toward the NUMA node where they are accessed. Trades migration overhead (latency spike) for long-term bandwidth locality.
throughput latency
Memory
kswapd / direct reclaim
kswapd proactively reclaims pages in background; direct reclaim is synchronous and occurs in the allocating task context. Their relative rate determines allocation latency.
latency throughput
Network
TCP pacing + BBR
BBR estimates bottleneck bandwidth and RTprop to pace sends. Avoids buffer-bloat, reducing RTT at the cost of some peak throughput on shallow-buffered paths.
latency throughput

03

Cross-subsystem policy relationships

structural dependencies
Policy A Policy B Relationship type Mechanism
CFS / EEVDF ⟷ kswapd coupled kswapd is scheduled as a kernel thread under CFS; memory pressure raises kswapd priority, competing with latency-sensitive tasks for CPU
BFQ ⟷ CFS / EEVDF shared model BFQ's budget-based fairness mirrors CFS vruntime logic; both use per-entity trees with a common fair-share abstraction
NUMA balancing ⟷ CFS task placement coupled NUMA fault handler hints to the scheduler to migrate the task alongside its pages; CFS load balancer can undo this if CPU imbalance dominates
RT throttling ⟷ mq-deadline tension RT tasks driving I/O can starve mq-deadline's normal queue; if RT bandwidth is capped, deadline expiry in the I/O layer may trigger before the task is scheduled
TCP BBR / pacing ⟷ CFS timeslice tension BBR's pacing timer fires via hrtimer; if CFS schedules the softirq thread late, pacing gaps widen, inflating measured RTprop and mistuning the BBR model
Page writeback ⟷ BFQ / mq-deadline coupled Writeback threads submit I/O tagged as background; both BFQ and mq-deadline deprioritise these — but their background queue rules differ, affecting dirty-page lag differently
PELT load tracking ⟷ cpufreq / EAS shared model Energy-Aware Scheduling reads CFS per-entity utilisation signals to select OPP; the same PELT signal drives both frequency scaling and task placement decisions
kswapd / direct reclaim ⟷ Kyber token buckets tension Under memory pressure, direct reclaim submits synchronous read I/O; Kyber's read token bucket may be depleted by unrelated foreground I/O, blocking reclaim and extending allocation latency
